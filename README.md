# NCR-Ride-SQL-Analysis
This project contains a comprehensive SQL analysis of NCR Ride trip data.
The objective is to explore, clean, and analyze the dataset to uncover insights about ride patterns, customer behavior, payments, and trip details.

All queries were performed in MySQL, and each question includes:

The SQL query
The output (exported as CSV)
A short summary of insights

#**Objectives of the Project**

Understand key trip metrics
Analyze ride patterns
Identify customer trends
Explore cancellations, payments, and fare amounts
Extract actionable insights using SQL

#**SQL Concepts Used**

This project uses a wide range of SQL topics:
SELECT, WHERE, ORDER BY, LIMIT
Aggregate functions: COUNT, SUM, AVG, MIN, MAX
GROUP BY & HAVING
DISTINCT
Pattern matching using LIKE
Date functions
Conditional logic using CASE
Sorting and filtering
Basic data exploration

#**Key Insights**

~The majority of rides are completed, showing high reliability.
~Online payments make up a significant share.
~Premium rides are fewer but generate higher revenue.
~Peak activity occurs in November 2024.
~Customers with higher ride frequency show consistent payment patterns.

#**Tools Used**

MySQL 8.0
MySQL Workbench
Microsoft Excel (for output exports)
